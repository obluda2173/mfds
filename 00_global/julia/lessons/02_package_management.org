* General
In the Julia programming language, =Manifest.toml= and =Project.toml= are two critical files that handle package management. They work together to ensure that code runs successfully and—crucially—is reproducible on other machines.

- *Analogy*:
  To understand the difference intuitively, think of visiting a restaurant.

  - =Project.toml= *is the Menu*: It tells you broadly what you want (e.g., "I want the Doner"). It does not care where exactly the tomatoes come from or exactly which chef cooked it, just that it fulfills the requirement of being a "doner".

  - =Manifest.toml= *is the Receipt*: This records exactly what happened after you ordered. It lists the exact cost, the time of transaction, the specific table you sat at, and the server's ID. It is a precise snapshot of that specific event.

* =Project.toml= (The Abstract Definition)
This file describes the identity of your project and its direct requirements. It is meant to be readable by humans.

- *What does it contain?*
  - *Metadata*: The name of your project, its unique ID (UUID), and version number.
  - *Direct Dependencies*: A list of packages you explicitly added (e.g., if you worked with /Plots/, it goes here).
  - *Compatibility Bounds*: Rules regarding which versions are allowed (e.g., "I can use any version of Plots as long as it is version 1.0 or higher").

- *Role*: It tells Julia, "This project needs DataFrames and CSV to work."

* =Manifest.toml= (The Concrete Snapshot)
This file is automatically generated by the computer. It describes the exact state of the package environment.

- *What does it contain?*
  - *The Full Tree*: It lists not just the packages you added, but the packages those packages depend on (recursive dependencies).
  - *Exact Versions*: It locks down the specific version numbers, the git tree hash (the precise source code commit), and the location of every single package installed.

- *Role*: It allows someone else (or you, 6 months later) to recreate the exact same environment, byte-for-byte. If =Project.toml= says "I need DataFrames," =Manifest.toml= says "I installed DataFrames v1.3.4, which relies on DataAPI v1.9.0 and Tables v1.6.0."

* Conclusion: The Workflow
You should almost *never* edit the TOML files manually. Instead, we use the Package Manager to generate them.

Here are the two ways to do this.

** Method 1: The Interactive Mode (Standard)
This is the most common way to manage packages while you are working.

*** Scenario A: Starting a new project (The Creator)
1. Open your terminal (or the terminal inside VS Code).
2. Navigate to your project folder:
   #+BEGIN_SRC sh
   cd path/to/my_project
   #+END_SRC
3. Start Julia by typing =julia= and pressing Enter. You should see the green =julia>= prompt.
4. Press the right bracket key =]=.
   /Notice that the prompt changes color and now says/ =(@v1.10) pkg>=. You are now in Package Mode.
5. Create a local environment by typing:
   #+BEGIN_SRC julia
   activate .
   #+END_SRC
   /The prompt will change to the name of your folder, e.g.,/ =(my_project) pkg>=.
6. Install packages:
   #+BEGIN_SRC julia
   add Plots DataFrames
   #+END_SRC

*** Scenario B: Running existing code (The User)
If you download a folder that already has a =Project.toml= and =Manifest.toml=:
1. Open terminal, navigate to the folder, start Julia, and press =]= to enter Package Mode.
2. Activate the project:
   #+BEGIN_SRC julia
   activate .
   #+END_SRC
3. Run the magic command:
   #+BEGIN_SRC julia
   instantiate
   #+END_SRC
   /This reads the TOML files and automatically downloads the exact versions of all packages needed./

** Method 2: The Code Method (Alternative)
If you find switching modes confusing, you can use standard Julia code commands instead. You can run these lines in the REPL or write them at the very top of your script file.

#+BEGIN_SRC julia
using Pkg

# 1. Activate the environment in the current folder (".")
Pkg.activate(".")

# 2. If it is a new project, add packages:
Pkg.add("Plots")
Pkg.add("DataFrames")

# OR

# 3. If you downloaded someone else's project, install dependencies:
Pkg.instantiate()
#+END_SRC
