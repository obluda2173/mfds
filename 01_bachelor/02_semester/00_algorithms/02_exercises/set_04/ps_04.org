* Problem 36 :random_sampling:random_permutation:
** Task
#+begin_quote
Professor Gallup wants to create a random sample of the set $\{1, \ldots, n\}$, that is, an $m$ element subset $S$, where $1 \leq m \leq n$, such that each $m$ subset is equally likely to be created. One way is to set $A[i] = i$, for $i = 1, 2, \ldots, n$, call RANDOMLY-PERMUTE($A$), and then take just the first $m$ array elements. Show that this indeed produces a random sample.
#+end_quote

** Solution

Fix $T \subseteq \{1, \ldots, n\}$ with $|T| = m$.

RANDOMLY-PERMUTE yields each permutation with probability $1/n!$. Number of permutations whose first $m$ positions contain exactly $T$:

- arrange $T$ in positions $1, \ldots, m$: $m!$ ways,
- arrange $\{1, \ldots, n\} \setminus T$ in positions $m+1, \ldots, n$: $(n-m)!$ ways.

\[
\Pr[S = T] = \frac{m!(n-m)!}{n!} = \frac{1}{\binom{n}{m}}.
\]

Independent of $T$, so all $\binom{n}{m}$ subsets are equiprobable. $\blacksquare$

*** Numerical check

For $n = 10$, $m = 3$: $\Pr[S = T] = \frac{3!\,7!}{10!} = \frac{1}{\binom{10}{3}} = \frac{1}{120} \approx 0.00833$.

#+begin_src julia :results value
using Random
let
    n, m = 10, 3
    target = Set(1:m)               # fix an arbitrary 3-subset
    epochs = 1_000_000
    hits = 0
    for _ in 1:epochs
        sample = Set(shuffle(1:n)[1:m])
        hits += (sample == target)
    end
    hits / epochs                   # should be ≈ 1/120 ≈ 0.00833
end
#+end_src

* Problem 38 :random_permutation:randomized_algorithm:uniform_distribution:
** Task
#+begin_quote
Professor Knievel suggests the procedure PERMUTE-BY-CYCLE to generate a uniform random permutation. Show that each element $A[i]$ has a $1/n$ probability of winding up in any particular position in $B$. Then show that Professor Knievel is mistaken by showing that the resulting permutation is not uniformly random.

1:  procedure PERMUTE-BY-CYCLE($A, n$)
2:    let $B[1 : n]$ be a new array
3:    $\text{offset} = \text{RANDOM}(1, n)$
4:    for $i = 1$ to $n$ do
5:      $\text{dest} = i + \text{offset}$
6:      if $\text{dest} > n$ then
7:         $\text{dest} = \text{dest} - n$
8:    end if
9:    $B[\text{dest}] = A[i]$
10:   end for
11:   return $B$
12: end procedure
#+end_quote

** Solution

*** (a)

For a fixed $i$ and target position $j$, $A[i]$ lands at $j$ iff $i + \text{offset} \equiv j  \mod n$. Exactly one offset in $\{1, \ldots, n\}$ satisfies this. Since the offset is uniform on $\{1, \ldots, n\}$, probability is $\frac{1}{n}$.

*** (b)

The algorithm is fully determined by the choice of $\text{offset} \in \{1, \ldots, n\}$, so it produces at most $n$ distinct permutations. But $\{1, \ldots, n\}$ admits $n!$ permutations, and $n < n!$ for $n \geq 3$. Hence at least $n! - n$ permutations occur with probability $0$, so the distribution is not uniform.

* Problem 39 :random_permutation:randomized_algorithm:
** Task
#+begin_quote
Professor Kelp decides to write a procedure that produces at random any permutation except the identity permutation, in which every element ends up where it started. He proposes the procedure PERMUTE-WITHOUT-IDENTITY. Does this procedure do what Professor Kelp intends?

1: procedure PERMUTE-WITHOUT-IDENTITY($A, n$)
2:   for $i = 1$ to $n - 1$ do
3:     swap $A[i]$ with $A[\text{RANDOM}(i + 1, n)]$
4:   end for
5: end procedure
#+end_quote

** Solution

*** (a)

Yes. At iteration $i = 1$, the algorithm swaps $A[1]$ with $A[\text{RANDOM}(2, n)]$, so $A[1]$ is displaced. Hence position $1$ cannot retain its original element, and the identity permutation is never produced.

*** (b)

No — the distribution is not uniform over the $n! - 1$ non-identity permutations.

The algorithm performs $n - 1$ iterations; iteration $i$ draws from a range of size $n - i$. Total number of outcome sequences:

\[
\prod_{i=1}^{n-1}(n - i) = (n-1)!.
\]

For $n \geq 3$, $(n-1)! < n! - 1$, so at least $n! - 1 - (n-1)!$ non-identity permutations are unreachable and occur with probability $0$. $\blacksquare$

* Problem 40 :random_permutation:randomized_algorithm:uniform_distribution:
** Task
#+begin_quote
Consider the PERMUTE-WITH-ALL, which instead of swapping element $A[i]$ with a random element from the subarray $A[i : n]$, swaps it with a random element from anywhere in the array. Does PERMUTE-WITH-ALL produce a uniform random permutation? Why or why not?

1: procedure PERMUTE-WITH-ALL($A, n$)
2:   for $i = 1$ to $n$ do
3:     swap $A[i]$ with $A[\text{RANDOM}(1, n)]$
4:   end for
5: end procedure
#+end_quote

** Solution

No. The algorithm runs $n$ iterations, each independently choosing an index from $\{1, \ldots, n\}$, giving $n^n$ equally likely execution paths, each with probability $1/n^n$.

If the resulting distribution over the $n!$ permutations were uniform, each permutation would be produced by exactly $n^n / n!$ paths. But the number of paths producing a given permutation is a non-negative integer, so uniformity requires $n! \mid n^n$.

For $n = 3$: $n^n / n! = 27 / 6 = 9/2 \notin \mathbb{Z}$, so $3! \nmid 3^3$. Hence the distribution is not uniform. $\blacksquare$

* Problem 41 :heap:binary_tree:height:
** Task
#+begin_quote
What are the minimum and maximum numbers of elements in a heap of height $h$? Show that an $n$ element heap has height $\lfloor \lg n \rfloor$.
#+end_quote

** Solution

*** (a)

Maximum number - $2^{h+1} - 1$, minimum - $2^h$.

*** (b)

A heap of height $h$ with $n$ elements satisfies, by (a),

\begin{align*}
2^h \le n &\le 2^{h+1} - 1 \\
h \le \lg n &\le \lg (2^{h+1} - 1) < h + 1 \\
h \le \lg n &< h + 1
\end{align*}

Hence $h = \lfloor \lg n \rfloor$. $\blacksquare$

* Problem 42 :heap:max_heap:
** Task
#+begin_quote
Consider a max-heap, in which all elements are distinct. Where can the smallest element reside? At which levels in a max-heap might the $k$ th largest element reside, for $2 \leq k \leq \left\lfloor \frac{n}{2} \right\rfloor$?
#+end_quote

** Solution

*** (a)

Leaves. In a max-heap any internal node exceeds its children, so the minimum cannot have any. Array indices $\lfloor n/2 \rfloor + 1, \ldots, n$.

*** (b)

\[
\lceil \lg k \rceil \le \ell \le \lfloor \lg n \rfloor.
\]

Upper: $\ell \leq \lfloor \lg n \rfloor$ (heap height); $k \leq \lfloor n/2 \rfloor$ keeps the $k$ th largest off the leaf-only level.

Lower: the $k - 1$ larger elements must fit in levels $0, \ldots, \ell - 1$, which hold at most $2^\ell - 1$ nodes:

\[
k - 1 \leq 2^\ell - 1 \iff \ell \geq \lceil \lg k \rceil. \qquad \blacksquare
\]

* Problem 43 :heap:min_heap:max_heap:
** Task
#+begin_quote
Is an array that is in sorted order a min-heap? Is the array with values
\[ [33, 19, 20, 15, 13, 10, 2, 13, 16, 12] \]
a max-heap?
#+end_quote

** Solution

*** (a)

Yes. For sorted ascending $A$ and any valid $i$: $i < 2i$ and $i < 2i + 1$, so $A[i] \leq A[2i]$ and $A[i] \leq A[2i+1]$. Min-heap property holds. Converse fails (a min-heap need not be sorted).

*** (b)

No. $A[4] = 15$ but $A[9] = 16$, violating $A[i] \geq A[2i+1]$.

#+begin_src julia :results output
let
    arr = [33, 19, 20, 15, 13, 10, 2, 13, 16, 12]
    n = length(arr)
    ok = true
    for i in 1:div(n, 2)
        if arr[i] < arr[2i] || (2i+1 ≤ n && arr[i] < arr[2i+1])
            ok = false
            break
        end
    end
    println(ok ? "max-heap" : "not max-heap")
end
#+end_src
