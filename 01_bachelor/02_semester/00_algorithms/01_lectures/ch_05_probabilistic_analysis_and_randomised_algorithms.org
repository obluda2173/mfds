* 5.1 The Hiring problem

- Interview $n$ candidates sequentially; hire whenever candidate $i$ beats current best
- Interview cost $c_i$, hiring cost $c_h$ (where $c_h \gg c_i$)
- Total cost: $O(c_i n + c_h m)$ where $m$ = number of hires
- Fixed $c_i n$ term → focus on analyzing $c_h m$

** Worst Case

- Candidates arrive in strictly increasing quality → hire all $n$
- Hiring cost: $O(c_h n)$

** Probabilistic Analysis

- Assume input drawn from a known distribution → compute *average-case running time*
- Hiring problem assumption: candidates arrive as a *uniform random permutation* — each of $n!$ orderings equally likely
- Requires knowledge/assumptions about input distribution; not always applicable

** Randomized Algorithms

- When input distribution is unknown: inject randomness into the algorithm itself
- Modified hiring: receive candidate list in advance, randomly permute interview order
- Behavior depends on input + output of RNG → *randomized* algorithm
- $\texttt{RANDOM}(a, b)$: returns integer in $[a, b]$ uniformly; independent calls
- Running time analyzed as *expected running time* (expectation over RNG), not over inputs

** Key Distinction

|                        | Source of randomness | Analysis type             |
|------------------------+----------------------+---------------------------|
| Probabilistic analysis | Input distribution   | Average-case running time |
| Randomized algorithm   | Algorithm's RNG      | Expected running time     |

* 5.2 Indicator random variables

** Definition

Given sample space $S$ and event $A$:

$$I\{A\} = \begin{cases} 1 & \text{if } A \text{ occurs} \\ 0 & \text{otherwise} \end{cases}$$

*** Lemma 5.1

$E[X_A] = \Pr\{A\}$

Proof: $E[I\{A\}] = 1 \cdot \Pr\{A\} + 0 \cdot \Pr\{\bar{A}\} = \Pr\{A\}$

** Key Property: Linearity of Expectation

$E\left[\sum_{i=1}^{n} X_i\right] = \sum_{i=1}^{n} E[X_i]$

- Holds *even when $X_i$ are dependent*
- Converts hard counting problems into sum of easy per-event probabilities

** Coin Flip Example

$X_i = I\{\text{flip } i \text{ is heads}\}$, $X = \sum X_i$

$$E[X] = \sum_{i=1}^{n} \frac{1}{2} = \frac{n}{2}$$

** Application to Hiring Problem

$X_i = I\{\text{candidate } i \text{ is hired}\}$, $X = \sum_{i=1}^{n} X_i$

- Candidate $i$ is hired iff best among first $i$ → $\Pr\{\text{hired}\} = 1/i$
- $E[X_i] = 1/i$

*** Expected number of hires

$$E[X] = \sum_{i=1}^{n} \frac{1}{i} = \ln n + O(1) = H_n$$

*** Lemma 5.2

Under uniform random permutation of candidates, HIRE-ASSISTANT has average-case hiring cost $O(c_h \ln n)$.

(vs. worst-case $O(c_h n)$)

*** Code

#+begin_src julia :results value
using Random

function count_hires(n)
    best = 0
    hires = 0
    for candidate in randperm(n)
        if candidate > best
            hires += 1
            best = candidate
        end
    end
    hires
end

function simulate(n, epochs)
    sum(count_hires(n) for _ in 1:epochs) / epochs
end

simulate(100, 1_000_000)
#+end_src

* 5.3 Randomized algorithms

** Randomized Hire-Assistant

- Don't *assume* input is uniformly random — *impose* it by permuting first
- Converts average-case guarantee (over inputs) into expected-time guarantee (over RNG, for *any* input)
- No adversarial input can trigger worst-case: bad performance requires unlucky RNG

#+begin_example
RANDOMIZED-HIRE-ASSISTANT(n)
1  randomly permute the list of candidates
2  HIRE-ASSISTANT(n)
#+end_example

** Lemma 5.3

Expected hiring cost of RANDOMIZED-HIRE-ASSISTANT is $O(c_h \ln n)$.

Proof: Permuting produces identical situation to Lemma 5.2's probabilistic analysis.

** Lemma 5.2 vs 5.3

|                          | Lemma 5.2 (probabilistic)           | Lemma 5.3 (randomized)               |
|--------------------------+-------------------------------------+--------------------------------------|
| Assumption               | Input is uniform random permutation | None (any input)                     |
| Source of randomness     | Input distribution                  | Algorithm's RNG                      |
| Cost term                | Average-case $O(c_h \ln n)$         | Expected $O(c_h \ln n)$              |
| Deterministic per input? | Yes (same input → same hires)       | No (same input → varies across runs) |

** Randomly Permuting Arrays (Fisher-Yates)

#+begin_example
RANDOMLY-PERMUTE(A, n)
1  for i = 1 to n
2      swap A[i] with A[RANDOM(i, n)]
#+end_example

- In-place, $\Theta(n)$ time
- Goal: each of $n!$ permutations with probability $1/n!$

** Correctness (Lemma 5.4)

Loop invariant: before iteration $i$, each possible $(i-1)$ permutation appears in $A[1:i-1]$ with probability $(n - i + 1)!/n!$.

- *Init*: $i = 1$ → $A[1:0]$ is empty → trivially holds (probability $n!/n! = 1$)
- *Maintenance*: Want $\Pr\{\langle x_1, \ldots, x_i \rangle \text{ in } A[1:i]\}  = (n-i)!/n!$
  - $E_1$: first $i-1$ iterations produced $\langle x_1, \ldots, x_{i-1} \rangle$ → $\Pr\{E_1\} = (n-i+1)!/n!$
  - $E_2$: iteration $i$ places $x_i$ in $A[i]$ → $\Pr\{E_2 \mid E_1\} = 1/(n-i+1)$
  - $\Pr\{E_2 \cap E_1\} = \frac{1}{n-i+1} \cdot \frac{(n-i+1)!}{n!} = \frac{(n-i)!}{n!}$ ✓
- *Termination*: $i = n+1$ → $A[1:n]$ is any $n$ permutation with probability $0!/n! = 1/n!$

** Subtlety

Showing each element lands in each position with probability $1/n$ is *not sufficient* to prove uniform random permutation (Exercise 5.3-4).
