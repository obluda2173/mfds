* Summary

The lecture began by recalling the two requirements for algorithm correctness established in the previous session: (1) the algorithm must halt, and (2) it must produce the correct output on all inputs. Both properties were then demonstrated for insertion sort.

The halting argument was handled first. Since insertion sort is non-recursive and contains only loops, the only risk of non-termination is an infinite loop. The outer =for= loop runs from $i = 2$ to $n$ and its index is never modified inside the body, so it terminates after $n - 1$ iterations. The inner =while= loop decrements $j$ by one in every iteration, and because $j$ starts at a positive value and the natural numbers are well-ordered, $j$ must eventually reach zero, forcing the loop condition to fail.

With halting established, the lecture turned to correctness of the output using the method of loop invariants. The concept was introduced in general: a loop invariant is a property $P(i)$ of the loop variable that is shown to hold before the loop begins (initialization), to be preserved across each iteration (maintenance), and to yield a useful conclusion when the loop terminates (termination). The analogy to mathematical induction was made explicit.

For insertion sort the chosen invariant is: at the start of iteration $i$ of the outer =for= loop, the sub-array $A[1..i-1]$ is a sorted permutation of the elements originally in $A[1..i-1]$. Initialization is trivial because $A[1..1]$ is a single element. For maintenance, the lecturer argued that the inner =while= loop shifts elements within $A[1..i]$ without disturbing their relative order and inserts the key into its correct position, so $A[1..i]$ becomes sorted. Two sub-cases were distinguished depending on whether $j$ reaches zero or the loop exits because $A[j] \leq \text{key}$. At termination, $i = n+1$ (or equivalently the loop has completed with $i = n$), so the invariant gives that $A[1..n]$ is sorted --- exactly the desired output.

The lecture then shifted to running-time analysis. Each line of insertion sort was assigned a cost constant $c_k$ and an execution count. The total running time was expressed as
\[ T(n) = c_1 n + c_2(n-1) + c_4(n-1) + c_5 \sum_{i=2}^{n} t_i + c_6 \sum_{i=2}^{n}(t_i - 1) + c_7 \sum_{i=2}^{n}(t_i - 1) + c_8(n-1), \]
where $t_i$ denotes the number of times the =while= loop test is executed during the $i$-th iteration of the =for= loop. The lecturer explained that these constants are unknown because we do not specify the exact CPU and its instruction-cycle costs; the assumption is that each line takes a fixed number of cycles per execution. In the best case (already sorted input), $t_i = 1$ for all $i$ and $T(n)$ is linear. In the worst case (reverse-sorted input), $t_i = i$ and $T(n)$ is quadratic.

To avoid carrying around unwieldy expressions with many constants, the lecture introduced asymptotic notation. The five standard notations --- $O$, $\Omega$, $\Theta$, $o$, $\omega$ --- were defined. The running time of insertion sort was classified as $O(n^2)$ and $\Omega(n)$, but not $\Theta$ of any single function because the best and worst cases have different growth rates. The historical remark was made that these are called Landau symbols, though most were popularised by Donald Knuth.

Finally, merge sort was introduced as a divide-and-conquer algorithm. The idea is to split the array into two halves, recursively sort each half, and merge the sorted halves by repeatedly selecting the smaller of the two front elements. The recursion bottoms out at single-element arrays. The merging procedure was described in detail: two index pointers walk through the sorted halves, always copying the smaller element into the output array, and when one half is exhausted the remainder of the other is appended.

*Key takeaways:* (1) Correctness proofs require showing halting and correct output. (2) Loop invariants are the standard tool for proving correctness of iterative algorithms and mirror mathematical induction. (3) Line-by-line cost analysis yields an exact expression for running time, but asymptotic notation ($O$, $\Omega$, $\Theta$, $o$, $\omega$) abstracts away constants and lower-order terms. (4) Insertion sort is $O(n^2)$ worst-case and $\Omega(n)$ best-case. (5) Merge sort uses the divide-and-conquer paradigm; its analysis was deferred to the next lecture.

* Definitions & Formal Concepts

** 1. Correct Algorithm
An algorithm is /correct/ if it satisfies two conditions: (a) it halts on every input, and (b) it produces the correct output for every input.

Intuition: a correct algorithm always finishes and always gives the right answer.

** 2. Loop Invariant
A /loop invariant/ for a loop with variable $i$ is a property $P(i)$ together with proofs of three clauses:
1. *Initialization:* $P(i)$ holds before the first iteration of the loop.
2. *Maintenance:* If $P(i)$ holds before an iteration, then $P(i')$ holds before the next iteration (where $i'$ is the updated loop variable).
3. *Termination:* When the loop exits, the invariant yields a useful property about the algorithm's output.

Intuition: a loop invariant is mathematical induction applied to a loop --- it tracks a property that is preserved at every step and delivers the desired conclusion at the end.

** 3. Insertion Sort Loop Invariant
For the outer =for= loop of insertion sort (with loop variable $i$ running from $2$ to $n$), the invariant is:

At the start of iteration $i$, the sub-array $A[1..i-1]$ consists of the elements originally in $A[1..i-1]$, rearranged in sorted (ascending) order.

Intuition: the first $i - 1$ slots always hold a sorted version of whatever was originally there.

** 4. Running Time of Insertion Sort
The total running time is
\[ T(n) = c_1 n + c_2(n-1) + c_4(n-1) + c_5 \sum_{i=2}^{n} t_i + c_6 \sum_{i=2}^{n}(t_i - 1) + c_7 \sum_{i=2}^{n}(t_i - 1) + c_8(n-1), \]
where $c_k$ are line-cost constants and $t_i$ is the number of =while=-loop test executions in the $i$-th =for=-loop iteration.

- *Best case* ($t_i = 1$): $T(n)$ is linear, i.e.\ $T(n) = an + b$ for constants $a, b$.
- *Worst case* ($t_i = i$): $T(n)$ is quadratic, i.e.\ $T(n) = an^2 + bn + c$ for constants $a, b, c$.

Intuition: the inner loop does no work when the array is already sorted, but does maximal work when the array is reverse-sorted.

** 5. Big-$O$ Notation (Asymptotic Upper Bound)
$f(n) = O(g(n))$ if there exist constants $c > 0$ and $n_0 \in \mathbb{N}$ such that
\[ 0 \leq f(n) \leq c \cdot g(n) \quad \text{for all } n \geq n_0. \]

Intuition: $f$ grows no faster than $g$, up to a constant factor, for large $n$.

** 6. Big-$\Omega$ Notation (Asymptotic Lower Bound)
$f(n) = \Omega(g(n))$ if there exist constants $c > 0$ and $n_0 \in \mathbb{N}$ such that
\[ 0 \leq c \cdot g(n) \leq f(n) \quad \text{for all } n \geq n_0. \]

Intuition: $f$ grows at least as fast as $g$, up to a constant factor, for large $n$.

** 7. Big-$\Theta$ Notation (Asymptotically Tight Bound)
$f(n) = \Theta(g(n))$ if there exist constants $c_1, c_2 > 0$ and $n_0 \in \mathbb{N}$ such that
\[ 0 \leq c_1 \cdot g(n) \leq f(n) \leq c_2 \cdot g(n) \quad \text{for all } n \geq n_0. \]

Intuition: $f$ grows at the same rate as $g$, sandwiched between two constant multiples.

** 8. Little-$o$ Notation (Strict Asymptotic Upper Bound)
$f(n) = o(g(n))$ if for every $\varepsilon > 0$ there exists $n_0 \in \mathbb{N}$ such that
\[ 0 \leq f(n) \leq \varepsilon \cdot g(n) \quad \text{for all } n \geq n_0. \]

Intuition: $f$ is asymptotically negligible compared to $g$.

** 9. Little-$\omega$ Notation (Strict Asymptotic Lower Bound)
$f(n) = \omega(g(n))$ if for every $R > 0$ there exists $n_0 \in \mathbb{N}$ such that
\[ R \cdot g(n) \leq f(n) \quad \text{for all } n \geq n_0, \quad f(n) \geq 0. \]

Intuition: $f$ grows asymptotically faster than any constant multiple of $g$.

** 10. Divide and Conquer
A /divide and conquer/ algorithm solves a problem by (1) dividing it into smaller sub-problems of the same type, (2) solving the sub-problems recursively, and (3) combining (conquering) the sub-solutions into a solution of the original problem.

Intuition: break the problem in half, solve each half, then stitch the results together. [Formulation was informal.]

** 11. Merge Sort
Merge sort is a divide-and-conquer sorting algorithm:
1. If the array has size 1, return (base case).
2. Split the array into two halves of size $\lfloor n/2 \rfloor$ and $\lceil n/2 \rceil$.
3. Recursively sort each half.
4. Merge the two sorted halves by repeatedly selecting the smaller front element.

Intuition: sorting is reduced to merging, and merging two sorted arrays is cheap because only the front elements need to be compared.

* Proofs

** Insertion Sort Halts

*Claim.* Insertion sort halts on every input of size $n \geq 1$.

*Proof method.* Case analysis on the loop structure, using the well-ordering of $\mathbb{N}$.

*Argument.*

Step 1. The algorithm is non-recursive and consists of elementary operations plus two loops, so the only possible source of non-termination is an infinite loop.

Step 2 (Outer loop). The =for= loop runs $i$ from $2$ to $n$. The variable $i$ is never modified inside the loop body. Therefore the loop executes exactly $n - 1$ iterations and terminates.

Step 3 (Inner loop). The =while= loop has condition $j > 0 \;\text{and}\; A[j] > \text{key}$. Within the loop body, $j$ is decremented by $1$ (line 7). If $j \leq 0$ at entry, the loop does not execute. If $j > 0$ at entry, then since $j$ decreases by $1$ each iteration and the natural numbers are well-ordered (every non-empty subset has a least element), $j$ must reach $0$ in finitely many steps, at which point the loop condition fails.

Step 4. Since both loops terminate and all other instructions are elementary, the algorithm halts. $\square$

** Insertion Sort Produces Correct Output (Loop Invariant Proof)

*Claim.* After insertion sort terminates, $A[1..n]$ is a sorted permutation of the original input.

*Proof method.* Loop invariant on the outer =for= loop.

*Loop invariant $P(i)$:* At the start of iteration $i$, the sub-array $A[1..i-1]$ is a sorted permutation of the original $A[1..i-1]$.

*Initialization* ($i = 2$). Before the first iteration, the sub-array $A[1..1]$ contains a single element, which is trivially sorted and is the element originally at position $1$.

*Maintenance.* Assume $P(i)$ holds: $A[1..i-1]$ is sorted and contains exactly the original elements from positions $1$ to $i-1$. We must show that after the $i$-th iteration, $A[1..i]$ is sorted and contains exactly the original elements from positions $1$ to $i$.

1. The algorithm sets $\text{key} = A[i]$ and $j = i - 1$.
2. The =while= loop only accesses indices in $\{1, \dots, i\}$: it reads $A[j]$ for $j \leq i-1$ and writes to $A[j+1]$ for $j+1 \leq i$. Therefore only elements from the original $A[1..i]$ are involved, and no elements outside this range are touched.
3. The =while= loop shifts elements: when $A[j] > \text{key}$, it sets $A[j+1] \leftarrow A[j]$ and decrements $j$. This copies each element at most one position to the right, preserving the relative order of elements in $A[1..i-1]$.
4. The loop terminates in one of two cases:
   - *Case 1: $j = 0$.* Every element in $A[1..i-1]$ was strictly greater than $\text{key}$, so $\text{key}$ is the smallest. It is placed at $A[1]$. Since the other elements retain their sorted order (shifted right by one), $A[1..i]$ is sorted.
   - *Case 2: $j > 0$ and $A[j] \leq \text{key}$.* The element $A[j]$ is $\leq \text{key}$, while the old $A[j+1]$ (now shifted to $A[j+2]$) is $> \text{key}$. Placing $\text{key}$ at $A[j+1]$ gives $A[j] \leq A[j+1] \leq A[j+2]$, and the rest of the array remains sorted.
5. In both cases, $A[1..i]$ is a sorted permutation of the original elements from positions $1$ to $i$. Thus $P(i+1)$ holds before the next iteration.

*Termination* ($i = n + 1$, i.e.\ the loop has completed its last iteration with $i = n$). By the invariant, $A[1..n]$ is a sorted permutation of the original $A[1..n]$. This is the desired output. $\square$

* Exam Preparation

** Recall

*Q1.* State the two conditions required for an algorithm to be correct.

:ANSWER:
(a) The algorithm halts on every input. (b) The algorithm produces the correct output on every input.
:END:

*Q2.* State the three clauses of a loop invariant proof.

:ANSWER:
Initialization (the property holds before the first iteration), Maintenance (if the property holds before an iteration it holds before the next), and Termination (when the loop ends the property yields the desired conclusion).
:END:

*Q3.* Write the formal definition of $f(n) = O(g(n))$.

:ANSWER:
There exist constants $c > 0$ and $n_0 \in \mathbb{N}$ such that $0 \leq f(n) \leq c \cdot g(n)$ for all $n \geq n_0$.
:END:

*Q4.* What is the best-case and worst-case asymptotic running time of insertion sort?

:ANSWER:
Best case: $\Theta(n)$ (input already sorted). Worst case: $\Theta(n^2)$ (input sorted in reverse).
:END:

*Q5.* Fill in the blank: $f(n) = o(g(n))$ means that for every ___ $> 0$, there exists $n_0$ such that $f(n) \leq$ ___ for all $n \geq n_0$.

:ANSWER:
For every $\varepsilon > 0$, there exists $n_0$ such that $f(n) \leq \varepsilon \cdot g(n)$ for all $n \geq n_0$.
:END:

** Comprehension

*Q6.* Explain why the =while= loop in insertion sort is guaranteed to terminate, but the argument requires more care than for the =for= loop.

:ANSWER:
The =for= loop has a fixed iteration count ($n - 1$ steps) because its index $i$ is never modified in the body. The =while= loop, by contrast, has a data-dependent condition ($j > 0$ and $A[j] > \text{key}$). To argue termination we must observe that $j$ is decremented by exactly $1$ each iteration and appeal to the well-ordering of $\mathbb{N}$: since $j$ starts positive and strictly decreases, it must reach $0$ in finitely many steps.
:END:

*Q7.* Insertion sort's running time is $O(n^2)$ and $\Omega(n)$, but not $\Theta$ of any single function. Explain why.

:ANSWER:
$\Theta(g(n))$ requires that the running time is both $O(g(n))$ and $\Omega(g(n))$ for the /same/ function $g$. Insertion sort's best case is linear and its worst case is quadratic, so there is no single $g$ that simultaneously serves as both a tight upper and tight lower bound across all inputs.
:END:

*Q8.* Trace through the merge step of merge sort on the two sorted sub-arrays $L = [1, 4, 7]$ and $R = [2, 5, 6]$. List the comparisons made and the resulting merged array.

:ANSWER:
Compare $L[1]=1$ vs $R[1]=2$: take $1$. Compare $L[2]=4$ vs $R[1]=2$: take $2$. Compare $L[2]=4$ vs $R[2]=5$: take $4$. Compare $L[3]=7$ vs $R[2]=5$: take $5$. Compare $L[3]=7$ vs $R[3]=6$: take $6$. $R$ exhausted, copy $L[3]=7$. Result: $[1, 2, 4, 5, 6, 7]$.
:END:

*Q9.* Distinguish $O(g(n))$ from $o(g(n))$ in terms of the role of the constant.

:ANSWER:
In $O(g(n))$, we require /some/ constant $c > 0$ such that $f(n) \leq c \cdot g(n)$ for large $n$. In $o(g(n))$, we require this for /every/ $\varepsilon > 0$, no matter how small. So $o$ is a strictly stronger condition: $f$ must be negligible relative to $g$, not merely bounded by a constant multiple.
:END:

** Proof-level

*Q10.* Prove that $3n^2 + 5n + 2 = O(n^2)$.

:ANSWER:
We need $c > 0$ and $n_0$ such that $3n^2 + 5n + 2 \leq c \cdot n^2$ for all $n \geq n_0$. For $n \geq 1$, $5n \leq 5n^2$ and $2 \leq 2n^2$, so $3n^2 + 5n + 2 \leq 3n^2 + 5n^2 + 2n^2 = 10n^2$. Choose $c = 10$ and $n_0 = 1$. $\square$
:END:

*Q11.* Let $A$ be an array in which every element is at most $k$ positions away from its final sorted position (with $k$ a fixed constant independent of $n$). Argue that the running time of insertion sort on such an input is $O(n)$.

:ANSWER:
In insertion sort, $t_i$ (the number of =while=-loop iterations in the $i$-th =for=-loop pass) equals one plus the number of positions element $A[i]$ must move left. If every element is at most $k$ positions from its sorted position, then $t_i \leq k + 1$ for all $i$. The sums $\sum_{i=2}^{n} t_i \leq (k+1)(n-1) = O(n)$, and similarly for the $t_i - 1$ sums. All other terms in $T(n)$ are $O(n)$. Since $k$ is a constant, $T(n) = O(n)$. $\square$
:END:

*Q12.* Reproduce the loop invariant proof that insertion sort produces a sorted array. (State the invariant, then prove initialization, maintenance, and termination.)

:ANSWER:
/Invariant $P(i)$:/ At the start of iteration $i$, $A[1..i-1]$ is a sorted permutation of the original $A[1..i-1]$.

/Initialization/ ($i=2$): $A[1..1]$ has one element, which is trivially sorted.

/Maintenance:/ Assume $P(i)$. The =while= loop shifts elements $> \text{key}$ one position right within $A[1..i]$ and places $\text{key}$ in the vacated slot. Two cases: (a) $j$ reaches $0$, meaning $\text{key}$ is the minimum and is placed at $A[1]$; (b) $A[j] \leq \text{key}$, so $\text{key}$ is placed at $A[j+1]$ between $A[j]$ and the shifted $A[j+2]$. In both cases $A[1..i]$ is sorted and contains exactly the original elements from positions $1$ to $i$, establishing $P(i+1)$.

/Termination:/ The loop ends after $i = n$, so $P(n+1)$ holds: $A[1..n]$ is a sorted permutation of the original input. $\square$
:END:

*Q13.* Prove from the definition that if $f(n) = O(g(n))$ and $g(n) = O(h(n))$, then $f(n) = O(h(n))$.

:ANSWER:
By hypothesis, there exist $c_1, n_1$ with $f(n) \leq c_1 g(n)$ for $n \geq n_1$, and $c_2, n_2$ with $g(n) \leq c_2 h(n)$ for $n \geq n_2$. Set $c = c_1 c_2$ and $n_0 = \max(n_1, n_2)$. Then for $n \geq n_0$:
\[ f(n) \leq c_1 g(n) \leq c_1 c_2 h(n) = c \cdot h(n). \]
Hence $f(n) = O(h(n))$. $\square$
:END:
