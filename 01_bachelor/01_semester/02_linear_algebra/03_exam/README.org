* OVERVIEW

This document distills strategic insights from preparing for Prof X's Linear Algebra exam across two attempts (Feb 2026 and Mar 2026). It is intended as a reusable playbook for future students taking the same course. The methodology is empirical: I analysed past papers, identified structural templates, and made predictions вҖ” most of which turned out correct.

* THE PROFESSOR'S EXAM TEMPLATE

** Fixed Structure (40 points, 6 problems)

Prof X uses a rigid template. The content rotates, but the skeleton does not.

| Problem | Type                                | Points | Style                                   |
|---------+-------------------------------------+--------+-----------------------------------------|
|       1 | Pure definition recall              |      4 | "Provide the definition of X"           |
|       2 | Map theory (define + check + prove) |      5 | 5 sub-questions, 1 point each (mostly)  |
|       3 | LU factorization                    |      7 | "Compute using LU (with pivoting)"      |
|       4 | Matrix inverse via row ops          |      6 | "Compute the inverse using row ops"     |

|       6 | Mixed short problems                |      8 | 3-4 sub-questions, varying point values |

** What Rotates Between Exam Dates

*** Problem 1: Axiomatic Definitions (4pt)

The professor cycles through foundational axiomatic definitions:

| Exam Date   | Definition Asked        |
|-------------+-------------------------|
| 1st attempt | Field (9 axioms)        |
| 2nd attempt | Vector Space (8 axioms) |

*Rule*: Never repeats across consecutive dates. The next candidate after these two would be the definition of a linear map, or potentially the definition of a field.

*Preparation strategy*: Memorize the FULL SYMBOLIC FORM of every axiom. Listing axiom names scores 0. You need the actual formulas.

*** Problem 2: Map Theory (5pt)

The professor pairs complementary concepts across attempts:

| Sub-question | 1st attempt             | 2nd attempt                     |
|--------------+-------------------------+---------------------------------|
| (a)          | Define f(A) (image)     | Define fвҒ»В№(B) (preimage)        |
| (b)          | Define injective        | Define surjective               |
| (c)          | Is xвҒҙ injective? (в„қвӮҠвҶ’в„қ) | Is xвҒҙ surjective? (в„қвӮҠвҶ’в„қ)        |
| (d)+(e)      | Define composition      | Define identity map             |
|              | Prove associativity     | Prove uniqueness of inverse map |

*Rule*: The professor mirrors concepts. If he asks injective, next
time he asks surjective. If he asks f(A), next time fвҒ»В№(B). The
concrete check (c) reuses the SAME FUNCTION with the complementary
property.

*Key definitions to have ready*:
- f(A), fвҒ»В№(B), injective, surjective, bijective
- Composition, identity map, inverse map
- Small proofs: associativity of composition, uniqueness of inverse,
  "composition of injectives is injective", "composition of surjectives
  is surjective"

*** Problem 3 & 4: Computation (13pt)

These are stable. Always 3Г—3 matrices over в„қ.

| Problem | 1st attempt     | 2nd attempt     |
|---------+-----------------+-----------------|
|       3 | LU with pivot   | LU with pivot   |
|       4 | Inverse via ops | Inverse via ops |

*Preparation strategy*: Practice 3-4 LU problems and 3-4 inverse
problems. Focus on arithmetic discipline вҖ” sign errors cascade. The
professor uses "nice" numbers but the intermediate fractions can still
trip you up.

*Critical note on fields*: The Data Science exam uses в„қ and в„Ӯ. The
historical past papers (different bachelor) use в„ӨвӮ… and в„ӨвӮҮ. Be aware
of this if using past papers for practice вҖ” the mechanics are the same
but modular arithmetic adds complexity.

*** Problem 5: The Big Proof (10pt)

This is the highest-value single question. The professor pairs
specific definition sets with specific proofs:

| Definition Package              | Paired Proof                  |
|---------------------------------+-------------------------------|
| Kernel, image, rank             | Steinitz Exchange (III.1.47)  |
| Kernel, image, rank             | Rank-Nullity (IV.1.10)        |
| Sum, direct sum, complement     | Dim Theorem for Sums (IV.1.4) |
| Subspace, sum, direct sum, comp | Dim Theorem for Sums (IV.1.4) |

*Rule*: The definition package PREDICTS the proof. If you see "define
kernel, image, rank" you know the proof will be about linear maps. If
you see "define sum, direct sum, complement" the proof will be IV.1.4.

*Critical insight*: The professor explicitly communicates which
theorems are in the exam pool. When new theorems are added (as happened
with IV.1.4, IV.1.10, IV.1.15 for the second attempt), recently tested
theorems become unlikely. Use this to prioritise.

*Proof priority for the two-attempt cycle*:

| Attempt | What appeared                        |
|---------+--------------------------------------|
| 1st     | Steinitz Exchange Theorem (III.1.47) |
| 2nd     | Dimension Theorem for Sums (IV.1.4)  |

If a 3rd attempt existed, Rank-Nullity (IV.1.10) or Steinitz (III.1.47) would be the most likely candidates.

*** Problem 6: Mixed Short Problems (8pt)

This block has the most variation but follows discoverable patterns.

**** Sub (a): Subspace Proof (2-3pt)

ALWAYS a "prove this set is a subspace" question. The specific sets rotate:

| Exam        | Set to check |                                |
|-------------+--------------+--------------------------------|
| 1st attempt | {p вҲҲ в„қ[X]    | p(0) = p(1) = 0}               |
| 2nd attempt | {v вҲҲ V       | h(v) = v} (fixed points)       |
| PP 1st date | {p вҲҲ в„Ӯ[X]    | p'(i) = 0}                     |
| PP 2nd date | {v вҲҲ V       | h(v) = (w,w)} (diagonal image) |
| PP 3rd date | {p вҲҲ в„қ[X]    | вҲ«вӮҖВ№ p(t)dt = 0}                |
| PP 4th date | {v вҲҲ V       | h(v) = v} (fixed points again) |

*Pattern*: When the main proof (P5) is about sums/complements, the
subspace question tends to use a linear map condition (fixed points,
preimage) rather than polynomial evaluation.

*Universal template*:
1. Zero vector: Show 0 вҲҲ S (plug 0 into condition, use f(0)=0)
2. Closure: Let v, w вҲҲ S and О» вҲҲ рқ”Ҫ. Show О»v + w вҲҲ S.
   - Expand what "v вҲҲ S" means
   - Compute condition applied to О»v + w
   - Use LINEARITY to split
   - Substitute known facts
   - Conclude

**** Sub (b)-(c): Quick Theory Questions (1pt each)

These rotate between a small set of types:
- "Does V contain a linearly independent set of k elements?" вҶ’ compare k with dim(V)
- "Does V contain a spanning set of k elements?" вҶ’ same dimension argument
- "Is this map linear?" вҶ’ check f(О»v + w) = О»f(v) + f(w) or find counterexample
- "Is this map injective/surjective?" вҶ’ Rank-Nullity or explicit check

*Key technique*: For spanning/independence questions, the answer
depends on comparing the number of elements with dim(V). A spanning
set needs вүҘ dim(V) elements. A linearly independent set needs вүӨ dim(V)
elements. Cite Steinitz for the bound.

**** Sub (d): Computation Over в„Ӯ (3pt)

On BOTH attempts, this was: "Compute the kernel of T: в„ӮВІ вҶ’ в„ӮВІ,
is T injective?"

| Attempt | Map                                       |
|---------+-------------------------------------------|
| 1st     | T(z,w) = ((1+5i)z + 2w, (8+i)z + (1-3i)w) |
| 2nd     | T(z,w) = ((2+4i)z + 2w, (5+6i)z + (3-i)w) |

*Template*:
1. Write the coefficient matrix A
2. Row reduce (complex arithmetic)
3. If rank = 2: ker = {0}, injective
4. If rank = 1: express free variable, write ker as span of one vector

*Warning*: The professor MAY switch to the "for which c вҲҲ в„Ӯ does
vector (c,...) lie in im(f)?" type, which appears in EVERY past paper
but was absent from both Data Science exams. This involves:
1. Set up 3Г—2 augmented matrix with parameter c
2. Row reduce to get [0 0 | g(c)]
3. Set g(c) = 0, solve, write as a + bi

* PROOF MEMORISATION STRATEGY

** The Three Proofs You Must Know

*** 1. Rank-Nullity Theorem (IV.1.10)

*Structure*: Basis extension вҶ’ show spanning вҶ’ show linear independence.

1. Choose basis {bвӮҒ,...,bвӮ–} for ker(f)
2. Extend to basis {bвӮҒ,...,bвӮҷ} for V
3. Claim: {f(bвӮ–вӮҠвӮҒ),...,f(bвӮҷ)} is basis of im(f)
4. Spanning: f(bвӮҒ)=...=f(bвӮ–)=0, so remaining images generate im(f)
5. Independence: Assume ОЈcбөўf(bбөў)=0, use linearity to get
   ОЈcбөўbбөў вҲҲ ker(f), express in kernel basis, invoke independence
   of {bвӮҒ,...,bвӮҷ} to force all coefficients = 0

*Common exam errors*: Wrong subscripts (writing bвӮ– instead of bвӮҷ),
writing "{bвӮҒ,...,bвӮ–} is basis of V" when it should be "{bвӮҒ,...,bвӮҷ}".

*** 2. Dimension Theorem for Sums (IV.1.4)

*Structure*: Shared basis extension вҶ’ show spanning вҶ’ show independence.

1. Choose basis Bбөў = {wвӮҒ,...,wвӮ–} for UвӮҒ вҲ© UвӮӮ
2. Extend to basis of UвӮҒ: add {uвӮҒ,...,uвӮҡ}
3. Extend to basis of UвӮӮ: add {vвӮҒ,...,vбө§}
4. Claim: B = {w's, u's, v's} is basis of UвӮҒ + UвӮӮ
5. Spanning: immediate from definition of sum
6. Independence: Assume ОЈОұбөўwбөў + ОЈОІвұјuвұј + ОЈОівӮ—vвӮ— = 0
   - Rearrange: ОЈОівӮ—vвӮ— = -(ОЈОұбөўwбөў + ОЈОІвұјuвұј) вҲҲ UвӮҒ
   - LHS вҲҲ UвӮӮ, so ОЈОівӮ—vвӮ— вҲҲ UвӮҒ вҲ© UвӮӮ, write as ОЈОјбөўwбөў
   - Independence of B_{UвӮӮ} forces ОівӮ— = 0
   - Substitute back, independence of B_{UвӮҒ} forces Оұбөў = ОІвұј = 0
7. Count: |B| = k + p + q = dim(UвӮҒ) + dim(UвӮӮ) - dim(UвӮҒ вҲ© UвӮӮ)

*** 3. Steinitz Exchange Theorem (III.1.47)

Required for first attempt. Lower priority for later attempts if new
theorems are added to the pool.

Requires: Exchange Lemma (III.1.46) + inductive argument.

** How to Memorise Proofs Effectively

Based on what worked in this preparation:

1. *Annotated rewriting*: Copy the proof, adding explanatory comments
   after each line in your own words.
2. *Personal review*: Rewrite from memory, check against original.
3. *Exam-style version*: Write it as you would on the exam вҖ” clean,
   structured, with numbered steps.
4. *From-memory drill*: Write it cold with no reference. Repeat until
   clean.

The progression from annotated вҶ’ personal вҶ’ exam-style вҶ’ from-memory
took 3-4 iterations per proof and was the most effective strategy used.

*** Why P6(b-d) Were Mispredicted

The professor chose to REPEAT the kernel+injective question from
exam_01 on the second attempt. This violated our assumption that he
would avoid repetition. Lesson: the no-repeat rule applies strongly
to the high-value slots (P1, P2, P5) but NOT to the low-value mixed
questions in P6, where he is willing to reuse question types.

The spanning set question (P6b) and linearity check (P6c) are from
a pool of ~6 short question types that rotate semi-randomly. These
are not individually predictable, but the POOL is small enough that
preparing all types is feasible.

** The Structural Pairing Insight

The single most valuable discovery: Prof X pairs definition packages
with proof theorems. When you see which definitions are asked, you
immediately know which proof is coming. This pairing is 100% reliable
across all observed exam dates.

Additionally, within Problem 2, the professor mirrors complementary
concepts between consecutive exam dates (injectiveвҶ”surjective,
f(A)вҶ”fвҒ»В№(B), compositionвҶ”identity).
