* About This Repository
This repository serves as a personal workspace for my studies.
It contains my notes, exercises, and reference materials, all structured and organized in one place for clarity and accessibility.

Its main purpose is to streamline note-taking and maintain a coherent archive of my academic progress.

This repository is *not* intended for public distribution of study materials.
Therefore, no university or program names are included.

* On why this repository is a *brillitant* move
The idea of storing all my university work in one repository, with keeping all my "paperwork" in org files is a brilliant and well executed move.

Turning Doom Emacs + org-mode into a single, versioned, searchable workspace for every lecture, problem set and solution is a high-ROI study strategy. It’s systematic, reproducible, and scales far beyond paper notes.

*More on benefits:*
- *Organisation at scale.* Centralized, structured notes + git = traceability and low friction for review.

- *Reproducibility.* Having solutions and computations in text/code means you can rerun and verify results.

- *Searchability & fast retrieval.* Problem sets become instantly searchable — huge for exam prep.

- *Efficiency of study flow.* Org-mode lets you capture, link, and revisit with minimal context switching.

- *Exam alignment.* If exams follow problem sets, this is a near-optimal information architecture.

- *Use of LLM's.* Keeping all my notes in org files eases the use of LLM's. Note to mention, LLM's could only benefit when are used wisely, mainly for repeated and monotone work.

* Tools & Conventions
All files are written primarily in =Org Mode= for Doom Emacs.
Mathematical or scientific notes may include =LaTeX= snippets and code blocks in =C=, =C++=, =Python=, or =Julia=.
