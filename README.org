- Lecture Notes

*My Goal:* To implement a new method of education, by making the learning process accumulative.

/Disclaimer: This is a personal project. It is not officially affiliated with the University of Vienna. No copyrighted university materials are hosted here-only original notes, code, and links to official resources./

* Primary Motivation

Whenever someone writes math on paper, they are essentially "drawing." It is a painting; written symbols lack semantic structure and are not reproducible. Most critically, paper notes suffer from knowledge decay-once the exam is over, the notebook is closed, and the knowledge becomes static and unsearchable.

This repository represents a fundamental shift from that traditional model. My goal is to treat my education not as a series of isolated semesters, but as a continuous software project.

- I want my work to be "real"-executable, queryable, and reusable. By digitizing the logic of mathematics and the syntax of programming:

- I stop "drawing" and start "encoding": Equations become standard LaTeX, capable of being parsed by both humans and machines (LLMs).

- I stop "reviewing" and start "running": Theorems are not just memorized; they are verified through embedded Julia/Python code.

- I stop "losing" and start "compounding": A script written for Linear Algebra in Semester 1 becomes a utility function for Machine Learning in Semester 4.

* The Workflow

This repository is built using *Doom Emacs* + *Org-Mode* + *LaTeX*. While you don't need Emacs to read the notes (GitHub renders them, plus I also provide rendered PDFs), this stack was chosen for specific strategic reasons:

1.  *Organization at Scale:* Centralized, structured notes + git = traceability. No more lost files.
2.  *Reproducibility:* Code (Python, C++, Julia) is embedded directly in the notes. I can run and verify math immediately.
3.  *Searchability:* `grep` my entire degree. Finding a specific theorem or code snippet takes milliseconds.
4.  *LLM-Ready:* Plain text files are the perfect input for LLMs. I use LLM to automate the monotonous work, leaving the complex thinking to us.
5.  *Exam Alignment:* The structure mirrors our problem sets, creating a near-optimal architecture for exam prep.
